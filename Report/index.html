1. Introduction

1.1 Purpose

GaMon is a tool to manage garbage collection in cities, manage reports from users about littering and initializing events for increasing awarness about the benefits of recycling.

2. Overall Description

2.1 Product Perspective

GaMon is a new, self-contained product, that is not based on any previous projects.

2.2 Product Functions

GaMon allows users to send data about the type of garbage thrown to a database from which specialized users can see the data in a centralized manner. Also the application allows the specialized users to send events about recycling. The administrator has access to the database trough a special page. He can also generate reports about garbage collection.

2.3 User Classes and Characteristics

The application has 3 types of users: user, superuser, administrator. The client can be either a citizen(user) which delivers his garbage to a bin or a waste management worker who needs to know where the most garbage congested areas are. The administrator is a priviledged user. He can acces the administrator page from where he can add/delete data from the database. Also he can obtain reports about the garbage accumulation in different areas, at different periods of time as CSV, HTML and PDF files.

3. System Features

3.1 Administration of database via the administration page

3.1.1 Description

The administrator is responsible for making active changes to the database.

3.1.2 Functional Requirements

The user has to input valid credentials in order to access the administrator page. The database must be up and running in order for the queries to take effect.

3.2 Searching and filtering results from the database

3.2.1 Description

The administrator uses the administrator page interface to filter and make queries to the database.

3.2.2 Functional Requirements

The database must be up and running in order for the queries to take effect.

3.3 Creating and downloading numeric reports and charts as HTML, CSV and PDF files

3.3.1 Description

This feature is responsible for generating reports as CSV, HTML, PDF files based on the search filters.

3.3.2 Functional Requirements

The database must be up and running in order for the queries to take effect.
